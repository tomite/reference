型名　　　　　解説　　　　　値
boolean       真偽値        true,false
byte          8ビット整数   -128から127
short         16ビット整数  -32768から32767
int           32ビット整数  -2147483648から2147483647
long          64ビット整数  -9223372036854775808lから
                            9223372036854775808l
char          UTF-16のコードポイント　'a'、'あ'、'言'、'\u03b1'
float         32ビット浮動小数点数  1.5f
double        64ビット浮動小数点数  123.456d

リファレンス型
リファレンス型にはクラス型、インターフェース型、型変数、配列型の4種類がある。
クラス型やインターフェース型はオブジェクトへの参照を表す。

変数の宣言
型 変数名

int number

変数の初期値
変数を宣言すると初期値が自動的に設定される。
リファレンス型の場合は全てnullで初期化される。プリミティブ型の場合は型によって初期値が異なる。プリミティブ型の初期値は以下の通り。

型　　　　　デフォルト値
byte        (byte) 0
short       (short) 0
int         0
long        0l
float       0.0f
double      0.0d
char        '\u0000' (null文字)
boolean     false

変数の初期値を設定できる
型 変数名 = 値
int number = 100

ラッパークラス
プリミティブ型にはそれぞれ対応するラッパークラスが提供されている。
ラッパークラスはプリミティブ型の値を持っているオブジェクトである。

プリミティブクラス　　ラッパークラス
boolean               java.lang.Boolean
byte                  java.lang.Byte
short                 java.lang.Short
int                   java.lang.Integer
long                  java.lang.Long
char                  java.lang.Character
float                 java.lang.Float
double                java.lang.Double

ラッパークラスは、プリミティブ型ではなくリファレンス型が必要となるケースで使われる。例えば、ジェネリクスの型変数にはリファレンス型を指定する必要があるなど。

List<Integer> intList = new Arrays.asList(1, 2, 3)

コンパイラが自動的にプリミティブ型からラッパークラス型へ変換することをオートボクシングと呼ぶ。
また、逆にプリミティブ型が必要な場合はラッパークラスをプリミティブ型へ変換することをアンボクシングと呼ぶ。
